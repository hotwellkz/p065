# Исправление ошибки: "Доступ заблокирован: приложение не прошло проверку Google"

## Ошибка

```
Доступ заблокирован: приложение "Shorts AI Studio" не прошло проверку Google
Ошибка 403: access_denied
```

## Причина

OAuth Consent Screen находится в режиме **"Testing"**, и ваш email не добавлен в список тестовых пользователей.

## Решение

### Шаг 1: Откройте OAuth Consent Screen

1. Откройте [Google Cloud Console](https://console.cloud.google.com/)
2. Выберите проект `prompt-6a4fd`
3. Перейдите в **APIs & Services** → **OAuth consent screen**
   - Прямая ссылка: https://console.cloud.google.com/apis/credentials/consent?project=prompt-6a4fd

### Шаг 2: Добавьте тестового пользователя

1. Прокрутите вниз до раздела **"Test users"**
2. Нажмите **"+ ADD USERS"** (или **"+ ДОБАВИТЬ ПОЛЬЗОВАТЕЛЕЙ"**)
3. Введите email: `bibi7475000@gmail.com` (или другой email, который будет использовать приложение)
4. Нажмите **"ADD"** (или **"ДОБАВИТЬ"**)
5. Сохраните изменения

### Шаг 3: Попробуйте снова

После добавления email в список тестовых пользователей:

1. Откройте: `http://localhost:8080/api/auth/google?userId=wJVWf7qvuoXYaVJSZbEGpNHUtva2`
2. Или используйте HTML страницу: `http://localhost:8080/google-oauth.html`
3. Авторизуйтесь через Google

## Альтернатива: Публикация приложения

Если хотите, чтобы приложение было доступно для всех пользователей:

1. В OAuth Consent Screen измените **"Publishing status"** на **"In production"**
2. Нажмите **"SUBMIT FOR VERIFICATION"** (Отправить на проверку)
3. Заполните форму верификации
4. Дождитесь одобрения Google (может занять несколько дней)

**Примечание**: Для продакшена может потребоваться верификация приложения Google, что занимает время.

## Быстрое решение

Для быстрого тестирования:
1. Добавьте ваш email (`bibi7475000@gmail.com`) в список тестовых пользователей
2. Это займёт 1-2 минуты
3. После этого авторизация будет работать сразу

## Проверка

После добавления тестового пользователя:

1. Откройте OAuth Consent Screen
2. Проверьте, что ваш email есть в списке "Test users"
3. Попробуйте авторизоваться снова

Если проблема сохраняется:
- Убедитесь, что используете тот же email, который добавили в тестовые пользователи
- Проверьте, что OAuth Consent Screen сохранён
- Подождите 1-2 минуты после добавления (изменения могут применяться с задержкой)
